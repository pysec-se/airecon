[PIPELINE PHASE: RECON - Reconnaissance and Enumeration]

OBJECTIVE:
Map the attack surface and collect enough baseline intelligence to form exploit hypotheses.
Quality over quantity — a well-understood single endpoint is better than a shallow scan of 100 hosts.

KEY INTELLIGENCE TO COLLECT:
1. Live services: confirm which endpoints/hosts are reachable and responding.
2. Application routes: root path, login, API endpoints, admin panels, upload paths.
3. Technology signals: framework, language, server, version hints from headers/errors/responses.
4. Authentication surface: login forms, session tokens, API keys, cookie patterns.
5. Input vectors: query parameters, form fields, headers, cookies that accept user data.
6. Exposure clues: error messages, debug info, backup files, exposed configs.
7. Historical/passive: archived routes, certificate subdomains (for multi-host targets).

RECON COMPLETION CRITERIA — advance when ANY of these are satisfied:
- At least one live endpoint confirmed reachable and returning meaningful content.
- Application technology or framework identified from response headers/body.
- At least one route or parameter beyond the root path discovered.
- Authentication mechanism located (login form, token endpoint, cookie pattern).
- For single-endpoint CTF targets: initial probe complete → advance immediately to ANALYSIS/EXPLOIT.

APPROACH BY TARGET TYPE:

Single-endpoint / CTF (localhost, single service):
1. Probe root and common paths (/, /login, /api, /admin, /register, /dashboard, /api/v1/).
2. Read response headers and body — identify technology, auth mechanism, interesting parameters.
3. Note any session cookies or tokens returned.
4. Advance to EXPLOIT after basic surface map — do not over-enumerate a CTF target.

Multi-host / External assessment:
1. Passive OSINT first: search with dork operators (site:{target}, inurl:admin, filetype:env, intext:password).
2. Subdomain enumeration → validate live hosts with httpx/dnsx BEFORE any scanning.
3. Port scan ONLY validated live hosts.
4. Directory/route discovery on confirmed live services.
5. Persist artifacts in output/.

PHASE DISCIPLINE:
- Do not repeat the same discovery tool twice with identical parameters.
- If two independent approaches find no new data, advance to ANALYSIS.
- Never block on incomplete subdomain enumeration for CTF/single-service targets.
