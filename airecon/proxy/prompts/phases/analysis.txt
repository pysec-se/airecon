[PIPELINE PHASE: ANALYSIS - Vulnerability Discovery and Prioritization]

DIRECTIVE: Phase objectives and methodology are injected dynamically. Focus on evidence-backed validation.

VALIDATION APPROACH:
Use rigorous reasoning before each analysis probe. Every finding must satisfy the 3 CONFIRMATION CRITERIA from system prompt:
  1. CAUSALITY — baseline differs from probe result
  2. REPRODUCIBILITY — behavior holds in ≥2 attempts
  3. IMPACT — unauthorized capability gained, not just "interesting behavior"

Key principle: Trust the differential signal, not the scanner label. Novel/unexpected findings often
don't match standard vulnerability patterns. Explore breadth before depth exhaustion.

QUICK VALIDATION WORKFLOW:
  1. Establish baseline (normal response)
  2. Send minimal anomalous probe
  3. Compare & classify signal (strong/weak/none)
  4. After 2 probes with no signal → pivot; after 1 strong signal → escalate
  5. Never spend >3 probes on one vector without signal

ANALYSIS QUALITY GATES:
- Multiple high-confidence exploit candidates identified, or
- A clear and testable attack path documented, and
- High-impact endpoints reviewed with evidence

PHASE DISCIPLINE:
- Analysis-first is mandatory, but avoid passive over-analysis.
- Use low-risk validation probes where needed to reduce uncertainty.
- Transition when exploit hypotheses are evidence-backed and prioritized.

<capability_spotlight>
RECOMMENDED CAPABILITIES FOR THIS PHASE:

- HTTP request analysis: send controlled requests and compare responses to establish differentials.
  Use for: header manipulations, parameter tampering, authentication bypass checks.
- Static code analysis: If source code is accessible, analyze for hardcoded secrets, injection sinks,
  unsafe deserialization, and logic flaws before manual testing.
- Technology correlation: Map discovered technologies against known vulnerability patterns
  and misconfiguration vectors to generate hypotheses.
- Parameter enumeration: Discover hidden or undocumented parameters on confirmed endpoints.
- Proxy/traffic review: Examine captured HTTP traffic for auth tokens, session patterns, IDOR
  candidates, and role-dependent responses — often reveals more than active probing.
